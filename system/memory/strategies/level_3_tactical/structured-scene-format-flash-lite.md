# Structured Scene Format Separation for Flash-Lite Prompt Simplification

## Context

Real-world A/B test failure in Wall Following scenario revealed that Gemini Flash-Lite prioritizes qualitative pattern matching over quantitative guidance when both are embedded in the same markdown block:

- Scene contained both SCENE PERCEPTION (descriptive text) and CLEARANCE/PROGRESS/OPTIONS (numerical values)
- Model fell back to pattern matching within SCENE PERCEPTION
- Ignored numerical guidance entirely
- Issued 133 MOVE_BACKWARD commands despite structured data indicating safe forward motion
- Full Stack version suffered 133 collisions due to out-of-bounds reversals

Root cause: Flash-Lite (ultra-low-latency, token-constrained) has different attention patterns than full Claude models. It treats interleaved qualitative text as the primary signal and embeds numerical values within the text as secondary signals that get deprioritized.

## Steps

1. **Separate narrative from structure** — Create distinct markdown sections:
   ```
   ## DECISIONS REQUIRED (structured numerical guidance)
   - CLEARANCE_FORWARD: [X] cm
   - CLEARANCE_LEFT: [X] cm
   - CLEARANCE_RIGHT: [X] cm
   - PROGRESS: [X]% toward goal
   - OPTIONS: [FORWARD, LEFT, RIGHT, BACKWARD] — ranked by safety

   ## SCENE PERCEPTION (descriptive narrative)
   - [Qualitative description of what the camera sees]
   - [Wall positions, obstacles, general layout]
   ```

2. **Prioritize DECISIONS REQUIRED first** — this section appears before and is separated visually from narrative context

3. **Provide explicit ranked options** in DECISIONS REQUIRED:
   - Instead of just listing CLEARANCE values, rank the available actions
   - Example: `OPTIONS: [FORWARD safe, LEFT moderate, RIGHT blocked, BACKWARD available]`

4. **Use numerical precision** in DECISIONS REQUIRED:
   - CLEARANCE_FORWARD: 125 cm (not "space ahead is clear")
   - PROGRESS: 67% (not "making good progress")
   - Thresholds: FORWARD safe if >50cm, MODERATE if 20-50cm, BLOCKED if <20cm

5. **Keep SCENE PERCEPTION brief and non-prescriptive**:
   - Describe what is visible without decision guidance
   - Reserve decision guidance for DECISIONS REQUIRED section only

6. **Test with Flash-Lite inference**:
   - Verify model consistently chooses actions from OPTIONS ranking
   - Monitor telemetry: does model issue MOVE_BACKWARD when CLEARANCE_BACKWARD < CLEARANCE_FORWARD?
   - If not, model may still be pattern-matching; increase structural separation

## Negative Constraints

- HIGH: Do not interleave numerical values within narrative description text
- HIGH: Do not assume Flash-Lite will parse numerical fields if they are surrounded by qualitative context
- MEDIUM: Do not use ambiguous relative language (e.g., "nearby", "far") in DECISIONS REQUIRED section — use absolute numerical values
- MEDIUM: Do not put ranked OPTIONS in SCENE PERCEPTION; separate all decision guidance to DECISIONS REQUIRED block

## Notes

- This pattern was observed specifically with Gemini 2.0-flash (token-constrained variant)
- Larger models (Claude, GPT-4) handle interleaved structured/narrative reasonably well; Flash-Lite does not
- The Wall Following failure (133 MOVE_BACKWARD commands) suggests Flash-Lite defaulted to a learned pattern ("reverse if confused") rather than consulting the structured data
- Prompt simplification benefit: removing narrative from decision block reduces token count while improving clarity
- This strategy is complementary to vision-loop system prompt design; it applies to both tool-calling and text-completion modes
